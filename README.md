# Лабораторна робота №2: Контейнеризація Python-додатку
Цей проект демонструє створення консольного перекладача на Python, його запуск у віртуальному оточенні та розгортання всередині Docker-контейнера.
## Хід виконання:
### 1. Підготовка середовища
Робота виконувалася у хмарному середовищі GitHub Codespaces. Після створення репозиторію та запуску контейнера розробки було виконано наступні кроки:
* Створення віртуального оточення:
```
python3 -m venv Kashtanova-Krystyna
source Kashtanova-Krystyna/bin/activate
```
* Створено файл requirements.txt з бібліотекою googletrans==3.1.0a0 та встановлено її:
```
pip install -r requirements.txt
```
### 2. Розробка програми
Створено файл main.py, який містить три основні функції:
* LangDetect(txt) — визначає мову введеного тексту.
* CodeLang(lang) — конвертує назву мови у код (наприклад, "English" -> "en") і навпаки.
* TransLate(str, lang) — виконує переклад на обрану мову.
### 3. Контейнеризація (Docker)
Для перенесення програми в ізольоване середовище було створено Dockerfile:  

**Опис Dockerfile:**
- Використано базовий образ python:3.11-slim (Linux Debian).
- Створено робочу директорію /Kashtanova_Krystyna у корені системи.
- Скопійовано файли коду та встановлено бібліотеки.

**Збірка образу:**
```
docker build -t kashtanova_ke .
```
**Запуск програми в контейнері:**  
```
docker run kashtanova_ke
```
### 4. Робота з Git
Для чистоти репозиторію було створено файл .gitignore, щоб не завантажувати тимчасові файли (__pycache__) та папку віртуального оточення.  

Збереження змін на GitHub:
```
git add .
git commit -m "Final version of Lab 2"
git push origin main
```
## Як запустити цей проект
Якщо ви хочете перевірити роботу програми:
- Склонуйте репозиторій.
- Маючи встановлений Docker, виконайте:
```
docker build -t translator-app .
docker run translator-app
```
